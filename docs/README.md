## About the Diataxis Framework

The Diataxis framework helps organize documentation to serve different user needs:

```
                 Learning  |  Tasks
                ------------|----------
    Study        Tutorials  |  How-to
                ------------|----------
    Work         Reference  |  Explanation
```

This structure ensures:
- **Findability** - Users can quickly locate the information they need
- **Clarity** - Each document type has a clear purpose
- **Completeness** - All user needs are addressed
- **Maintainability** - Contributors know where to add new content

## Contributing to Documentation

When adding documentation:

1. **Tutorials** - Add learning-focused content that guides through a complete process
2. **How-To Guides** - Add task-focused guides that solve specific problems
3. **Reference** - Add technical specifications, API docs, or implementation plans
4. **Explanation** - Add conceptual content that explains "why" and provides context
